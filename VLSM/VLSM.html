    VLSM¶

    La técnica de VLSM (Variable Length Subnet Mask) permite dividir una red en subredes de diferentes tamaños, optimizando el uso de direcciones IP.

    Note

    En los ejercicios, se usará el criterio RFC 1878 La subred cero. Es decir, se permite zero-subnet (Subred #1 con todos los bits de red a cero) y all-ones-subnet (última subred con todos los bits de red a 1). Además usaremos la notación CIDR.

    Pasos para Distribuir Subredes con VLSM¶

    1. Identificar las Necesidades de Hosts: - Haz una lista de todas las subredes que necesitas y el número de hosts requeridos para cada una.

    2. Ordenar las Subredes: - Ordena las subredes en orden descendente según el número de hosts necesarios. Esto asegura que las subredes más grandes se asignen primero, evitando fragmentación. Aunque si que vamos a tener en cuenta posibilidad de crecimiento en ciertas subredes.

    3. Calcular la Máscara de Subred: - Para cada subred, calcula la máscara de subred necesaria usando la fórmula:

    \(2^h - 2 \geq \text{número de hosts}\) donde \( h \) es el número de bits de host.

    1. Asignar Direcciones IP: - Comienza con la primera subred y asigna el rango de direcciones IP adecuado. Luego, ajusta la máscara de subred y repite el proceso para las siguientes subredes.

    2. Actualizar la Tabla de Subredes:

    • Mantén una tabla con la siguiente información para cada subred:
      • Número de subred
      • Hosts requeridos
      • Hosts disponibles
      • Dirección de red
      • Máscara de subred
      • Primera IP utilizable
      • Última IP utilizable
      • Dirección de broadcast

    Ejemplo Práctico¶

    Supongamos que tienes una red 192.168.1.0/24 y necesitas crear subredes para los siguientes departamentos:

    • Departamento A: 50 hosts
    • Departamento B: 20 hosts
    • Departamento C: 10 hosts
    1. Ordenar Subredes:
    • Departamento A: 50 hosts
    • Departamento B: 20 hosts
    • Departamento C: 10 hosts
    1. Calcular Máscaras de Subred:
    • Departamento A: Necesita al menos 50 hosts, por lo que la máscara será /26 (64 direcciones, 62 hosts utilizables).
    • Departamento B: Necesita al menos 20 hosts, por lo que la máscara será /27 (32 direcciones, 30 hosts utilizables).
    • Departamento C: Necesita al menos 10 hosts, por lo que la máscara será /28 (16 direcciones, 14 hosts utilizables).

    Ejercicio: Diseño de Subredes con VLSM y Seguridad en VLANs¶

    Una institución educativa ha adquirido el bloque de direcciones 172.16.0.0/22 y necesita dividirlo en subredes eficientes usando VLSM.

    Requerimientos de la red:¶

    1. Departamento de Informática: 110 hosts
    2. Departamento de Administración: 50 hosts
    3. Departamento de Profesorado: 30 hosts
    4. Zona de Aulas (Red WiFi para alumnos): 250 hosts
    5. Conexión entre switches de core y distribución: 3 enlaces punto a punto (cada uno requiere 2 direcciones IP)

    Resolución¶

    Paso 1: Determinar el tamaño de cada subred¶

    Usamos VLSM para asignar direcciones de manera eficiente. Recordemos que una subred necesita al menos hosts + 2 direcciones (una para la dirección de red y otra para el broadcast).

    • Zona de Aulas WIFI: 250 hosts → 2^8 = 256 direcciones (/24)
    • Departamento de Informática: 110 hosts → 2^7 = 128 direcciones (/25)
    • Departamento de Administración: 50 hosts → 2^6 = 64 direcciones (/26)
    • Departamento de Profesorado: 30 hosts → 2^6 = 64 direcciones (/26)
    • Enlaces punto a punto: 3 enlaces, cada uno requiere 2 IPs → 2^2 = 4 direcciones por enlace (/30)

    Paso 3: Cálculo de eficiencia¶

    • Total de direcciones disponibles: 1024 (/22 = 2^10)
    • Total de direcciones usadas: 640
    • Porcentaje de uso: (640 / 1024) * 100 = 62.5%
    • Direcciones desperdiciadas: 1024 - 640 = 384
